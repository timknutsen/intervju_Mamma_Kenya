# Story workflow

This folder now holds the active English editorial workflow.

## Current state

Right now the active story layer is English-only.

Active files live in `stories/en/`.

Earlier Norwegian drafting has been intentionally archived and is not part of the current working baseline.

The current review site is already built from these files into `docs/` for GitHub Pages deployment.

## Target model

The active master workflow is English-first.

The intended text stack is:

1. English interview-faithful version
2. English story-faithful version
3. English story-optimized version
4. Norwegian versions produced later from approved English layers

## What each version means

### English interview-faithful

- Closest possible cleaned version of the actual interview
- Still formatted as an interview
- Minimal restructuring
- Should feel very recognizable to your mother

### English story-faithful

- Tells the material as she told it
- Preserves her movement through memory, anecdote, contradiction, and tone
- More like memoir prose, but still very close to her storytelling voice

### English story-optimized

- Improves flow, chronology, transitions, and literary force
- May reorder and shape material more strongly
- Must still be traceable to the earlier layers

## Folder roles

- `stories/en/`: active master story layer
- `archive/legacy-editorial/`: archived Norwegian and other retired editorial material

## Near-term goal

The near-term goal is not a final book manuscript. It is to build a set of strong reviewable English text versions that can later be published on a website, likely via GitHub Pages, so your mother can read and respond.

Current next step:

- review the English interview-faithful version first
- collect corrections and recognizability feedback
- revise the story-faithful layer from that
- only then revise the story-optimized layer
